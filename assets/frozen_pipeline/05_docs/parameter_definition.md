# Parameter Definition

本文件定义本项目中所有主要表格、字段、参数含义、单位、来源、处理方式、输出位置和统计规则。

Codex 和 Python 脚本必须严格依据本文件生成结果表。不得自行改变核心字段名称和字段顺序。若确需新增字段，应写入日志，并在输出表末尾追加。

## 2. Data Source Priority

元数据读取优先级：

```text
sample_index.csv manual correction
filename parsing
folder path
error or warning
```

若文件名与 `sample_index.csv` 冲突：

1. 默认以 `sample_index.csv` 为准。
2. 必须记录到 `logs/run.log`。
3. 必须标记 `main_warning_flag = metadata_conflict`。
4. 不应中断全局分析。

原始波形文件处理优先级：

```text
raw waveform file
format detection
format-specific loading
normalized waveform CSV
feature extraction
classification
statistics
```

后续特征提取、绘图和统计必须基于：

```text
04_results/run_*/normalized_waveform/normalized_csv/
```

不得直接基于原始文件执行后续分析。

## 3. `sample_index.csv`

用途：建立每个原始波形文件到实验条件的映射。

字段顺序固定为：

```text
file_name,file_path,date,experiment_id,group_id,config_group,distance_m,load_condition,ground_condition,block_id,sample_index,included,manual_note
```

| Field | Meaning | Unit | Source | Processing Rule |
|---|---|---|---|---|
| `file_name` | 原始波形文件名 | none | 文件系统 | 自动读取 |
| `file_path` | 原始波形文件路径 | none | 文件系统 | 自动读取相对路径或绝对路径 |
| `date` | 实验日期 | YYYYMMDD | 文件名 | 从 `YYYYMMDD` 解析 |
| `experiment_id` | 实验编号 | none | 文件名 | 实验一固定为 `ML01` |
| `group_id` | 组合编号 | none | 自动生成 | 由配置、距离、负载、接地方式拼接 |
| `config_group` | 配置组 | none | 文件名 | `G1/G2/G3/G4` |
| `distance_m` | 接收距离 | m | 文件名 | 如 `D0p3m` 转换为 `0.3` |
| `load_condition` | 终端负载 | label | 文件名 | 如 `L50ohm` 转换为 `50ohm` |
| `ground_condition` | 接地方式 | none | 文件名 | 如 `Gshort` 转换为 `short` |
| `block_id` | 采集块号 | none | 文件名 | 如 `B01` |
| `sample_index` | 样本序号 | none | 文件名 | 如 `S001` |
| `included` | 是否纳入分析 | 0/1 | 用户或Codex | 默认 `1`，用户可改为 `0` |
| `manual_note` | 人工备注 | none | 用户 | 现场现象或补充说明 |

说明：

- `sample_index.csv` 可以先建空表，仅包含表头。
- Codex 扫描 `00_raw_csv/` 后应自动补全。
- 用户可手动修改 `included` 和 `manual_note`。
- Codex 不应覆盖用户已经填写的 `manual_note`。

## 4. `group_definition.csv`

用途：定义每组组合，控制组统计口径。

字段顺序固定为：

```text
group_id,config_group,distance_m,load_condition,ground_condition,expected_N,group_description,include_in_summary
```

| Field | Meaning | Unit | Source | Processing Rule |
|---|---|---|---|---|
| `group_id` | 组合编号 | none | 用户或Codex | 应与文件名解析生成规则一致 |
| `config_group` | 配置组 | none | 用户 | `G1/G2/G3/G4` |
| `distance_m` | 接收距离 | m | 用户 | 数值型，如 `0.3` |
| `load_condition` | 终端负载 | label | 用户 | 如 `50ohm`, `1Mohm`, `open` |
| `ground_condition` | 接地方式 | none | 用户 | 如 `short`, `long`, `coax`, `loop` |
| `expected_N` | 预期样本数 | count | 用户 | 用于核查缺失样本 |
| `group_description` | 组合说明 | none | 用户 | 人工说明 |
| `include_in_summary` | 是否纳入组统计 | 0/1 | 用户 | `1`纳入，`0`不纳入 |

说明：

- `group_definition.csv` 由用户预先创建。
- Codex 可读取，不应随意覆盖。
- 若 `sample_index.csv` 中存在 `group_definition.csv` 未定义的 `group_id`，应在日志中记录 warning，并标记 `undefined_group`。


| Field | Meaning | Unit | Source | Processing Rule |
|---|---|---|---|---|
| `run_id` | 分析批次编号 | none | Codex | 自动生成 |
| `file_id` | 单文件唯一编号 | none | Codex | 由 `group_id + block_id + sample_index` 生成 |
| `file_name` | 原始文件名 | none | 文件系统 | 自动读取 |
| `file_path` | 原始文件路径 | none | 文件系统 | 自动读取 |
| `file_extension` | 文件扩展名 | none | 文件系统 | 如 `.csv`, `.xlsx`, `.xls` |
| `detected_format` | Codex识别出的真实格式 | none | 格式检测 | 如 `true_text_csv`, `excel_xls_or_ole_binary` |
| `loader_used` | 实际读取器 | none | Codex | 如 `pandas_read_csv_auto`, `parse_sequence_volt_start_increment` |
| `parse_status` | 解析状态 | none | Codex | `success` 或 `error` |
| `time_column_mode` | 时间轴来源 | none | Codex | `direct_time_column`, `reconstructed_from_sequence`, `unknown` |
| `voltage_column_mode` | 电压列来源 | none | Codex | 如 `direct_voltage_column`, `Volt_column` |
| `start_time_s` | 起始时间 | s | 文件或计算 | 若无则 `NaN` |
| `time_increment_s` | 采样间隔 | s | 文件或计算 | 若无则由时间列中位差计算 |
| `sampling_rate_Hz` | 采样率 | Hz | 计算 | `1 / time_increment_s` |
| `record_length` | 数据点数 | count | 标准化数据 | 自动统计 |
| `error_message` | 错误信息 | none | Codex | 成功时为空 |
| `suggested_action` | 建议处理方式 | none | Codex | 成功时为空 |

允许的 `detected_format`：

```text

csv_with_preamble
excel_xlsx
excel_xls_or_ole_binary
mislabeled_excel_binary_with_csv_extension
unknown_binary
```

## 6. Normalized Waveform CSV

用途：将不同示波器、不同导出格式统一为标准波形输入。

位置：

```text
04_results/run_*/normalized_waveform/normalized_csv/{file_id}_normalized.csv
```

字段顺序固定为：

```text
time_s,voltage_V
```

| Field | Meaning | Unit | Source | Processing Rule |
|---|---|---|---|---|
| `time_s` | 标准化时间轴 | s | 原始文件或重建 | 必须单调递增 |
| `voltage_V` | 标准化电压 | V | 原始文件或单位转换 | 若原始为 mV，必须转换为 V |

若原始文件为示波器 Sequence-Volt-Start-Increment 格式：

```text
Row 0: X, CH1, Start, Increment
Row 1: Sequence, Volt, start_s, increment_s
Row 2+: sequence_index, voltage
```

则重建公式为：

```text
time_s = start_s + sequence_index × increment_s
voltage_V = voltage
sampling_rate_Hz = 1 / increment_s
```

## 14. Feature Extraction Definitions

### 14.1 `Apk`

```text
Apk = max(abs(v(t))) within analysis window
```

含义：主分析窗口内最大绝对幅值。

单位：

```text
V
```

---

### 14.2 `Amin`

```text
Amin = min(v(t)) within analysis window
```

单位：

```text
V
```

---

### 14.3 `App`

```text
App = max(v(t)) - min(v(t))
```

单位：

```text
V
```

---

### 14.4 `t_Apk_s`

```text
t_Apk_s = time point where abs(v(t)) reaches Apk
```

单位：

```text
s
```

---

### 14.5 `f1_MHz` and `A1`

`f1_MHz` 是设定频率搜索范围内，完成独立峰簇合并后幅值最大的清晰峰频率。  
`A1` 是该主频处的谱幅值。

单位：

```text
f1_MHz: MHz
A1: a.u.
```

---

### 14.6 `f2_MHz` and `A2`

`f2_MHz` 是排除 `f1` 所在独立峰簇后，幅值第二大的独立清晰峰频率。  
`A2` 是该第二峰处的谱幅值。

---

### 14.7 `PSR_dB`

```text
PSR_dB = 20 * log10(A1 / A2)
```

若 `A2 <= 0` 或无法找到第二峰：

```text
PSR_dB = NaN
main_warning_flag = no_second_peak
```

---

### 14.8 `narrowband_SNR_dB`

窄带 SNR 定义为信号频带功率与噪声频带功率之比：

```text
narrowband_SNR_dB = 10 * log10(P_signal_band / P_noise_band)
```

信号带宽和噪声频带由 `analysis_config.yaml` 控制。

---

### 14.9 `alpha`

`alpha` 为阻尼系数，来自主 burst 窗口内的阻尼正弦拟合。

拟合模型：

```text
v(t) = C + A * exp(-alpha * t_rel) * sin(2*pi*f*t_rel + phi)
```

其中 `t_rel` 为拟合窗口内归零后的相对时间。

单位：

```text
1/s
```

若拟合失败：

```text
alpha = NaN
fit_R2 = NaN
analysis_status = fit_failed
main_warning_flag = fit_failed
```

---

### 14.10 `rho`

`rho` 为模板相关系数，用于评估当前波形与 medoid 模板波形的一致性。

范围：

```text
-1 to 1
```

若模板不存在：

```text
rho = NaN
main_warning_flag = template_missing
```

---

### 14.11 `fit_R2`

拟合优度，建议范围：

```text
0 to 1
```

若拟合失败：

```text
fit_R2 = NaN
```

---

### 14.12 `fsrc_MHz`

`fsrc_MHz` 为参考源频率。  
默认从参考组：

```text
G3_D0p3m_L1Mohm_Gshort
```

中 A/B 类或 success 波形的 `f1_MHz` 中位数建立。

单位：

```text
MHz
```

---

### 14.13 `A_fsrc`

`A_fsrc` 为频谱中 `fsrc_MHz` 附近窗口的谱幅值。

单位：

```text
a.u.
```

---

### 14.14 `f1_to_fsrc_diff_percent`

```text
f1_to_fsrc_diff_percent = abs(f1_MHz - fsrc_MHz) / fsrc_MHz * 100
```

单位：

```text
%
```

---

### 14.15 `fsrc_trackable`

`fsrc_trackable` 表示当前波形中是否仍能追踪参考源频率附近的弱峰。

允许值：

```text
true
false
```

---

### 14.16 `f2_relation`

第二峰性质。

允许值：

```text
independent_peak
harmonic
noise
unknown
```

---

## 16. Statistics Definition

### 16.1 Median

```text
median = 中位数
```

---

### 16.2 IQR

```text
IQR = Q3 - Q1
```

---

### 16.3 CV

```text
CV_percent = sample_standard_deviation / mean × 100%
```

规则：

1. 标准差使用样本标准差。
2. 若均值接近 0，则 `CV_percent = NaN`。
3. 若 `CV_percent = NaN`，应在 `logs/run.log` 中记录 warning。

---

### 16.4 Quality Ratio

```text
excellent_ratio_percent = N_excellent / N_total × 100%
usable_ratio_percent = N_usable / N_total × 100%
suspicious_ratio_percent = N_suspicious / N_total × 100%
unusable_ratio_percent = N_unusable / N_total × 100%
pass_ratio_percent = N_pass / N_total × 100%
```
